// The Network Evidence Layer //
Why Network Forensics
Every attack traverses the network — even fileless, memory-only malware must communicate. Network evidence is often the only artifact that survives host cleanup, log deletion, or VM reversion. Mastering it closes the gap that host-based forensics leaves open.
Protocol Stack — Forensic Value
PACKET LAYERS & WHAT EACH REVEALS
Each OSI layer carries distinct forensic signals. A complete network investigation touches all of them.
L7 APP APPLICATION HTTP headers · DNS queries · TLS SNI · SMTP envelopes · SMB paths C2 URLs · phishing indicators · exfil data
L6/5 TLS/SESSION TLS version · cipher suite · SNI · certificate details · JA3 fingerprint Malware TLS profiles · self-signed certs
L4 TRANS TCP/UDP Source/dest ports · TCP flags · sequence numbers · connection state Port scanning · unusual port use · SYN floods
L3 NET IP Source/dest IPs · TTL · protocol · fragmentation · ICMP type C2 IPs · lateral movement paths · tunneling
L2 LINK ETHERNET MAC addresses · VLAN tags · ARP ARP poisoning · VLAN hopping

Zeek Scripting
CUSTOM ZEEK DETECTION SCRIPT
Write Zeek scripts to detect custom behavioral patterns. This script detects beaconing by tracking connection intervals per host.
ZEEK SCRIPT ## beacon_detect.zeek ## Detect C2 beaconing: regular intervals to same external IP module BeaconDetect; export { const beacon_threshold = 8 &redef; # min connections const jitter_tolerance = 0.2 &redef; # 20% jitter allowed } # Track connection times per src->dst pair global conn_times: table[addr, addr] of vector of time; event connection_established(c: connection) { local orig = c$id$orig_h; local resp = c$id$resp_h; # Only track external destinations if ( resp in Site::private_address_space ) return; if ( [orig, resp] !in conn_times ) conn_times[orig, resp] = vector(); conn_times[orig, resp] += network_time(); } event zeek_done() { for ( [src, dst] in conn_times ) { local times = conn_times[src, dst]; if ( |times| < beacon_threshold ) next; # Calculate inter-arrival time deltas local deltas: vector of double; for ( i in times ) { if ( i == 0 ) next; deltas += interval_to_double(times[i] - times[i-1]); } # Low coefficient of variation = regular = beacon local mean = mean(deltas); local stddev = stddev(deltas); local cv = stddev / mean; # coefficient of variation if ( cv < jitter_tolerance ) { NOTICE([$note=Weird::Activity, $msg=fmt("BEACON: %s→%s interval=%.1fs cv=%.3f count=%d", src, dst, mean, cv, |times|), $src=src, $dst=dst]); } } }

DNS Forensics
DNS is the most information-rich protocol in the network analyst's toolkit — and the most abused. DGA detection, DNS tunneling forensics, passive DNS correlation, and covert channel analysis.
DGA Detection
DETECTING DOMAIN GENERATION ALGORITHMS
DGA-generated domains have characteristic statistical properties — high entropy, unusual length distribution, and distinctive character n-gram patterns.
PYTHON import math, re from collections import Counter def domain_entropy(domain: str) -> float: """Shannon entropy of the second-level domain.""" sld = domain.split('.')[0] # e.g. 'k3mq9xzt4' from 'k3mq9xzt4.com' freq = Counter(sld) total = len(sld) return -sum(c/total * math.log2(c/total) for c in freq.values()) # Consonant-to-vowel ratio (DGA domains have high ratio) def consonant_ratio(domain: str) -> float: sld = domain.split('.')[0].lower() vowels = set('aeiou') consonants = [c for c in sld if c.isalpha() and c not in vowels] letters = [c for c in sld if c.isalpha()] return len(consonants)/len(letters) if letters else 0 def is_dga(domain: str) -> dict: sld = domain.split('.')[0] ent = domain_entropy(domain) cvr = consonant_ratio(domain) length = len(sld) has_digits = bool(re.search(r'\d', sld)) score = 0 reasons = [] if ent > 3.5: score += 2; reasons.append(f"high entropy={ent:.2f}") if cvr > 0.75: score += 2; reasons.append(f"high consonants={cvr:.2f}") if length > 15: score += 1; reasons.append(f"long label={length}") if has_digits: score += 1; reasons.append("contains digits") return {"domain": domain, "score": score, "dga_likely": score >= 4, "reasons": reasons} # Test against DNS log with open("dns.log") as f: for line in f: if line.startswith('#'): continue fields = line.split('\t') if len(fields) < 10: continue domain = fields[9] # query field result = is_dga(domain) if result["dga_likely"]: print(f"[DGA?] {domain} — {result['reasons']}")

// Encrypted Traffic Analysis //
TLS Forensics & JA3 Fingerprinting
Even encrypted traffic leaves fingerprints. JA3 hashes the TLS ClientHello parameters to identify the client implementation — and specific malware families have recognizable JA3 signatures.
JA3 — How It Works
TLS FINGERPRINTING METHODOLOGY
JA3 extracts five fields from the TLS ClientHello, concatenates them with delimiters, and MD5-hashes the result. Different client implementations (malware families, browsers, OS TLS stacks) produce different JA3 values.
PYTHON — JA3 CALCULATION import hashlib def compute_ja3( tls_version: int, # e.g. 771 (TLS 1.2) cipher_suites: list, # e.g. [49196, 49195, ...] extensions: list, # extension type IDs elliptic_curves: list, # supported groups ec_point_formats: list, # point format IDs ) -> str: """ JA3 = MD5(SSLVersion,Ciphers,Extensions,EllipticCurves,EllipticCurveFormats) Each list joined with '-', components joined with ',' GREASE values (0x?a?a) are excluded before hashing """ GREASE = {0x0a0a,0x1a1a,0x2a2a,0x3a3a,0x4a4a,0x5a5a, 0x6a6a,0x7a7a,0x8a8a,0x9a9a,0xaaaa,0xbaba, 0xcaca,0xdada,0xeaea,0xfafa} cs = "-".join(str(c) for c in cipher_suites if c not in GREASE) ext = "-".join(str(e) for e in extensions if e not in GREASE) ec = "-".join(str(c) for c in elliptic_curves if c not in GREASE) epf = "-".join(str(p) for p in ec_point_formats) raw = ",".join([str(tls_version), cs, ext, ec, epf]) return hashlib.md5(raw.encode()).hexdigest() # Example: Cobalt Strike default beacon JA3 # JA3 = 72a7c4f777d392aa0a9b06b6d20dc79f # Zeek extracts this automatically in ssl.log ja3 field

Statistical and behavioral methods for detecting C2 beaconing in network traffic — the attacker's most persistent network footprint. Periodicity analysis, byte volume patterns, and protocol anomalies.
Statistical Beacon Detection
PERIODICITY ANALYSIS — PYTHON
PYTHON import pandas as pd import numpy as np from scipy.stats import variation def detect_beacons( conn_log: str, min_connections: int = 10, max_cv: float = 0.3, # coefficient of variation threshold exclude_private: bool = True, ) -> pd.DataFrame: """ Analyze Zeek conn.log for beaconing behavior. Low CV (coefficient of variation) = regular intervals = beacon. """ df = pd.read_csv(conn_log, sep='\t', comment='#', low_memory=False, names=['ts','uid','orig_h','orig_p', 'resp_h','resp_p','proto', 'service','duration','orig_bytes', 'resp_bytes','conn_state']) df['ts'] = pd.to_numeric(df['ts'], errors='coerce') df = df.dropna(subset=['ts']) if exclude_private: private = r'^(10\.|192\.168\.|172\.(1[6-9]|2\d|3[01])\.|127\.)' df = df[~df['resp_h'].str.match(private, na=False)] results = [] grouped = df.groupby(['orig_h', 'resp_h', 'resp_p']) for (src, dst, dport), grp in grouped: if len(grp) < min_connections: continue times = grp['ts'].sort_values().values deltas = np.diff(times) if len(deltas) < 3: continue cv = variation(deltas) # stddev/mean — lower = more regular mean = np.mean(deltas) total_bytes = grp['orig_bytes'].sum() + grp['resp_bytes'].sum() if cv < max_cv: results.append({ 'src': src, 'dst': dst, 'port': dport, 'count': len(grp), 'interval': round(mean, 1), 'cv': round(cv, 4), 'bytes': int(total_bytes), }) out = pd.DataFrame(results) if len(out) == 0: return out return out.sort_values('cv') # most regular first beacons = detect_beacons("conn.log") print(beacons.to_string(index=False))

PYTHON import pandas as pd import networkx as nx def map_lateral_movement(conn_log: str, suspicious_ports=(445,135,3389,5985,22)): """Build lateral movement graph from conn.log.""" df = pd.read_csv(conn_log, sep='\t', comment='#', usecols=['ts','orig_h','resp_h','resp_p', 'orig_bytes','resp_bytes'], names=['ts','uid','orig_h','orig_p','resp_h','resp_p', 'proto','service','duration','orig_bytes','resp_bytes'], low_memory=False) # Filter to lateral movement ports only lateral = df[df['resp_p'].isin(suspicious_ports)] # Build directed graph G = nx.DiGraph() for _, row in lateral.iterrows(): G.add_edge(row['orig_h'], row['resp_h'], port=row['resp_p'], bytes=row['orig_bytes']+row['resp_bytes']) # Find pivot points (hosts that connect AND receive connections) pivots = [n for n in G.nodes() if G.in_degree(n) > 0 and G.out_degree(n) > 0] print(f"Hosts with lateral connections: {G.number_of_nodes()}") print(f"Lateral connections: {G.number_of_edges()}") print(f"\nPivot points (attacker likely used these):") for p in pivots: ins = list(G.predecessors(p)) outs = list(G.successors(p)) print(f" {p}: received from {ins} → moved to {outs}") return G G = map_lateral_movement("conn.log") # Visualize: nx.draw(G, with_labels=True) + matplotlib
